# 보안 정책 (Security Policy)

## 🛡️ 지원되는 버전

최상의 안전 표준을 유지하기 위해 주요 프로젝트의 최신 메이저 버전을 적극적으로 유지 관리합니다.

| 버전    | 지원 여부          |
| ------- | ------------------ |
| v1.0.x  | :white_check_mark: |
| < v1.0  | :x:                |

---

## 🚨 취약점 보고 절차 (Reporting a Vulnerability)

보안 취약점을 발견하신 경우, 아래의 **책임 있는 공개(Responsible Disclosure)** 프로세스를 따라주세요:

1.  **이메일 공개**: 상세 보고서를 [rheehose@rheehose.com](mailto:rheehose@rheehose.com)으로 보내주세요.
2.  **접수 확인**: 보고서 수신 후 **24시간 이내**에 회신드립니다.
3.  **평가**: **72시간 이내**에 예비 평가 결과를 제공합니다.
4.  **수정 및 출시**: 수정을 완료한 후 출시하며, 원하는 경우 보고자에게 공로를 표합니다 (Credits).

**취약점이 공식적으로 해결될 때까지 공개적으로 공개하지 말아 주시길 부탁드립니다.**

## 🗝️ 보안 채널

매우 민감한 취약점의 경우, 전체 보고서를 보내기 전에 이메일로 PGP 공개 키를 요청하실 수 있습니다.
