# Rhee Creative 프로젝트 기여 가이드라인

환영합니다! Rhee Creative의 프로젝트에 관심을 가져주셔서 감사합니다. 저희는 고품질의 미학적으로 뛰어난 소프트웨어를 만드는 데 전념하고 있습니다.

## 🚀 기여 방법 (How to Contribute)

1.  **이슈 찾기**: 열려 있는 이슈를 확인하거나 새로운 기능을 제안해 주세요.
2.  **포크 및 클론**:
    ```bash
    git clone https://github.com/hslcrb/[project-name].git
    cd [project-name]
    ```
3.  **브랜치 생성**:
    ```bash
    git checkout -b feat/your-exciting-feature
    ```
4.  **구현 및 테스트**: 코드를 작성하고 저희의 미학적 표준을 충족하는지 확인하세요.
5.  **PR 제출**: 변경 사항과 그 필요성에 대해 명확한 설명을 작성해 주세요.

---

## 📝 커밋 컨벤션 (Commit Conventions)

모든 커밋은 **한영 병기 (한국어 / 영어)** 형식을 따라야 합니다.
- 형식: `[타입]: [한국어 요약] / [영어 요약]`
- 예시: `feat: 새로운 UI 테마 추가 / Add new UI theme`

**주요 타입:**
- `feat`: 새로운 기능
- `fix`: 버그 수정
- `style`: 미학적 변경 (UI/UX)
- `docs`: 문서 업데이트
- `refactor`: 기능 변경 없는 코드 개선

## ✨ 미학적 가이드라인

-   **글래스모피즘(Glassmorphism)**: 백드롭 필터(블러, 채도)가 적용된 반투명 배경을 사용하세요.
-   **프루티거 에어로(Frutiger Aero)**: 광택 있는 질감, 활기찬 그라데이션, 인문주의적 유기적 형태에 집중하세요.
-   **미니멀리즘(Minimalism)**: 모든 픽셀은 목적이 있어야 합니다. 불필요한 요소를 제거하세요.

## 🛠️ 개발 표준

- **린팅(Linting)**: 커밋 전 관련 린팅 도구를 실행하세요.
- **테스트**: 핵심 모듈에서 회귀 버그가 없는지 확인하세요.
- **리뷰**: 모든 PR은 병합 전 최소 하나의 상세 리뷰를 거쳐야 합니다.
